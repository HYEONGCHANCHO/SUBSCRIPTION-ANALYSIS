# Ticket: 20260409-02-pdf-preprocessing-optimization

## 1. 개요
- **목적:** PDF 원본 전체를 AI에 전달하는 대신, 핵심 정보만 추출하여 전달함으로써 분석 속도 및 정확도 향상
- **배경:** 대용량 PDF 처리는 API 할당량 소모가 크고, 모델이 핵심 수치를 오판할 가능성이 있음

## 2. 현재 상태
- **방식:** PDF 파일 경로를 Gemini CLI에 직접 전달하여 전체 문맥 분석 의존
- **문제점:** 토큰 소모 과다, `MODEL_CAPACITY_EXHAUSTED` 에러 발생 빈도 높음, 분석 시간 지연

## 3. 계획
- [ ] **텍스트 전처리기 개발:** `pdf-parse`를 이용해 PDF에서 텍스트를 추출하고, 핵심 키워드(공급금액, 전용면적, 일정 등)가 포함된 문단만 필터링
- [ ] **AI 프롬프트 최적화:** 전체 문맥 대신 정제된 데이터 기반으로 JSON 결과를 도출하도록 프롬프트 수정
- [ ] **속도 개선:** 텍스트 기반 요청은 PDF 전체 분석보다 응답 속도가 3~5배 빠름
- [ ] **안정성 확보:** API 호출 간 대기 시간을 유지하되, 전체 작업 시간은 단축됨
