# 청약 공고 자동 분석 서비스 설계 (Subscription Analysis Service Design)

본 문서는 `.gemini/agents/` 및 `.gemini/skills/`의 지침을 기반으로 작성된 프로젝트 상세 설계서입니다.

## 1. 프로젝트 목표
- 로컬 환경에서 주요 청약 사이트(청약홈, LH, SH)의 공고 자동 수집 및 AI 분석.
- 사이트별 독립 모듈화 및 사용자 맞춤형 필터링 제공.

## 2. 기술 스택
- Playwright (Headless Mode), SQLite, Node.js (TypeScript).

## 3. 상세 아키텍처
- **Scraper Layer:** `IScraper` 인터페이스 기반 사이트별 전용 어댑터.
- **Storage Layer:** `downloads/[SiteName]/[YYYY]/[MM]/[DD]/[Title].pdf` 계층 구조.

## 4. 상세 구현 계획

### 4.1 청약홈 수집 (Verified)
- (기존 구현 완료)

### 4.2 LH 청약 수집 프로세스 (Phase 2-2)
1. **경기도 수집 단계:**
   - 지역 드롭다운에서 '경기도' 선택 후 검색.
   - **제외 키워드 필터링:** '부천', '평택', '안성', '구리', '남양주', '양주', '동두천', '하남', '김포', '인천', '파주', '청년 전세임대', '가정어린이집' 포함 시 제외.
   - 조건에 맞는 공고 상세 페이지 진입.
2. **서울특별시 수집 단계:**
   - 경기도 작업 완료 후 지역을 '서울특별시'로 변경하여 검색.
   - **제외 키워드 필터링:** '청년 전세임대' 포함 시 제외.
3. **상세 처리 및 다운로드:**
   - 상세 페이지 내 첨부파일 중 분석이 쉬운 **PDF**를 최우선으로 다운로드 (없을 경우 HWPX 등 차선책 선택).
   - 작업 완료 후 목록으로 돌아와 중복되지 않은 모든 공고 처리.
4. **계층적 저장:**
   - 경로: `backend/data/downloads/LH/[YYYY]/[MM]/[DD]/[공고명].pdf`

## 5. 검증 및 품질 관리
