# Clean Code & Refactoring Skill

## Guidance
- **Naming:** Use clear, descriptive, and consistent names for variables, functions, and files.
- **Functionality:** Functions should do one thing and do it well. Avoid long, complex logic.
- **DRY (Don't Repeat Yourself):** Extract common logic into reusable functions or components.
- **Comments:** Prefer self-documenting code. Use comments for 'Why', not 'What'.
- **Modern Syntax:** Utilize modern language features (ES6+, etc.) for concise and readable code.

## Workflow
- Refactor code before and after adding new features.
- Address technical debt incrementally.
- Eliminate dead code and redundant comments.
